## Overview

This is an MkDocs Material blog hosted at `https://blog.pollockweb.com/`. Content is written in Markdown and deployed to GitHub Pages via GitHub Actions on push to `main`. The GitHub repo is `cloudlabmp/project-blog`.

## Commands

**Local development:**
```bash
# Create and activate virtual environment (WSL)
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies (including social card imaging)
pip install -r requirements.txt
pip install "mkdocs-material[imaging]"

# Serve locally with live reload
mkdocs serve

# Build static site
mkdocs build

# Deploy to GitHub Pages manually
mkdocs gh-deploy --force
```

> Note: The `git-committers` and `git-revision-date-localized` plugins are currently `enabled: false` in `mkdocs.yml`. They require a `MKDOCS_GIT_COMMITTERS_APIKEY` environment variable (GitHub token). CI handles this via the build cache.

## Architecture

### Key Files
- `mkdocs.yml` — All site configuration: nav, theme, plugins, markdown extensions
- `requirements.txt` — Python dependencies for MkDocs and all plugins
- `hooks/socialmedia.py` — MkDocs hook that appends X/Facebook share buttons to any page using `template: blog-post.html`
- `overrides/main.html` — Extends base theme via Jinja2 block inheritance; provides `extrahead` block (Font Awesome, `article:author` OG tag) and `announce` block (social links bar)
- `docs/authors.yml` — Author profiles referenced in post front matter
- `.github/workflows/ci.yml` — CI: builds and deploys to `gh-pages` branch on push to `main`

### Content Structure
Blog posts live under `docs/blog/posts/` organised by topic subdirectory (e.g., `azure-posts/`, `ai-automation/`, `terraform-ai-monitoring/`). Each post typically gets its own subdirectory to co-locate images and supporting files alongside the `.md` file.

### Post Front Matter
All blog posts require this front matter format:
```yaml
---
title: "Post Title"
description: "Short description for SEO and RSS"
date:
  created: YYYY-MM-DD
  updated: YYYY-MM-DD
authors:
  - matthew
categories:
  - Azure
tags:
  - azure
  - relevant-tag
reading_time: "X min"
---
```

The `<!-- more -->` delimiter controls the RSS abstract cutoff and blog index excerpt.

### Plugins in Use
- `blog` — MkDocs Material blog engine (posts, tags, archive)
- `social` (with `cards: true`) — Auto-generates social preview card images (1200x630 PNG) and injects Open Graph + Twitter Card meta tags (`og:title`, `og:description`, `og:image`, `twitter:card`, etc.) into every page. Requires `mkdocs-material[imaging]` extra.
- `rss` — RSS feed from `blog/posts/**`
- `git-committers` + `git-revision-date-localized` — Show contributor info and dates (currently disabled locally; require git history and GitHub token)
- `glightbox` — Zoomable images

### Social / Open Graph Tags
OG tags are handled by two mechanisms:
1. **`social` plugin** (`cards: true` in `mkdocs.yml`) — generates `og:title`, `og:description`, `og:image`, `og:url`, `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image` automatically from page metadata
2. **`overrides/main.html`** (`extrahead` block) — adds `article:author` meta tag (not generated by the social plugin)

The `meta` plugin (Insiders-only) is NOT available — do not add it. The community `social` plugin handles OG tags on its own.

### Theme Overrides
`overrides/main.html` uses Jinja2 block inheritance (`{% extends "base.html" %}`). All custom head content must go inside `{% block extrahead %}`. Never place raw `<head>` tags outside the Jinja template inheritance — they will be injected before the base template and cause issues.

### Deployment
GitHub Actions (`ci.yml`) runs on push to `main`:
1. Checks out the repo with `actions/checkout@v4`
2. Sets up Python 3.12
3. Caches MkDocs Material assets (keyed by week number)
4. Installs `requirements.txt`
5. Runs `mkdocs build`
6. Injects `CNAME` file (`blog.pollockweb.com`) into `site/`
7. Deploys via `mkdocs gh-deploy --force` to the `gh-pages` branch

The custom domain `blog.pollockweb.com` is configured via `docs/CNAME`. Pushing to `main` triggers automatic deployment — no manual `gh-deploy` needed.

### LinkedIn / Social Sharing Notes
- After deploying changes that affect OG tags, re-scrape the URL using [LinkedIn Post Inspector](https://www.linkedin.com/post-inspector/) to bust the cache
- LinkedIn requires OG images to be at least 1200x627px — the social plugin generates 1200x630px cards
- LinkedIn may re-scrape existing posts automatically when OG tags change, but this is not guaranteed
